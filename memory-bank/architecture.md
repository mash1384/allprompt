## 주요 모듈 역할 및 관계

### 핵심 로직 모듈 (src/core/)

#### file_scanner.py
- **역할**: 파일 시스템 검색 및 파일 정보 수집
- **주요 기능**:
  - `scan_directory(root_path, follow_symlinks, include_hidden)`: 
    - 지정된 루트 경로부터 재귀적으로 모든 파일과 폴더를 스캔
    - 각 항목에 대해 경로, 상대 경로, 디렉토리 여부, 심볼릭 링크 여부, 숨김 여부 등의 정보를 수집
    - 권한 오류 등이 발생할 경우 오류 정보를 포함하여 접근 가능한 항목만 반환
    - `follow_symlinks` 옵션으로 심볼릭 링크를 따라갈지 결정
    - `include_hidden` 옵션으로 숨김 파일/폴더 포함 여부 결정
  - `is_hidden(path)`: 
    - OS에 독립적으로 파일/폴더의 숨김 여부 감지
    - Unix/Linux/Mac에서는 파일명이 '.'으로 시작하는지 확인
    - Windows에서는 시스템 API를 통해 파일 속성 확인
  - `is_binary_file(file_path)`: 
    - 파일의 앞부분을 읽어 널 바이트(0x00) 포함 여부로 텍스트/바이너리 판별
    - 파일이 비어있거나 존재하지 않는 경우 적절히 처리
    - 일반적인 바이너리 파일 확장자(.exe, .jpg, .pdf 등) 목록을 통한 빠른 식별
    - 확장자 기반 검사 후 바이트 검사를 통한 이중 확인 메커니즘 적용
  - `read_text_file(file_path)`: 
    - 텍스트 파일 내용을 문자열로 읽기
    - 다양한 인코딩을 체계적으로 시도 (UTF-8, UTF-8-sig, UTF-16, Latin-1, EUC-KR, CP949, Shift-JIS 등)
    - 파일 존재 여부, 바이너리 여부, 접근 권한, 인코딩 감지 실패 등 모든 오류 상황에 대한 상세한 오류 정보 제공
    - 오류 발생 시 단순 None 대신 구조화된 딕셔너리 반환 {'error': str, 'error_type': str, 'details': str}
- **의존 관계**: 독립적 모듈, 외부 의존성 없음
- **상세 설명**:
  - `pathlib`를 사용하여 파일 시스템 탐색
  - 심볼릭 링크 감지 및 처리
  - 파일 접근 권한 오류 처리
  - OS별 숨김 파일 속성 감지
  - 바이너리 파일 식별을 위한 다양한 전략 적용:
    - 확장자 기반 빠른 식별 (성능 최적화)
    - 파일 내용 바이트 검사 (정확성 보장)
    - 조합된 접근 방식으로 높은 정확도와 성능 제공

#### tokenizer.py
- **역할**: 텍스트를 LLM 모델 기준 토큰으로 변환 및 토큰 수 계산
- **주요 기능**:
  - `Tokenizer` 클래스: 토큰화 객체 관리
    - 다양한 OpenAI 모델 지원을 위한 `TOKENIZER_MODELS` 딕셔너리 관리
    - 각 모델별 인코딩 방식 및 최대 토큰 제한 정보 저장
    - 지원하지 않는 모델 요청 시 기본 모델(gpt-3.5-turbo)로 대체하는 안전 메커니즘
    - 모델 인코딩 초기화 오류에 대한 예외 처리
  - `count_tokens(text)`: 텍스트 토큰 수 계산
    - None 또는 빈 문자열 입력 시 0 반환
    - tiktoken 인코딩 객체를 사용하여 정확한 토큰 수 계산
    - 토큰화 과정의 예외 상황 처리 및 로깅
  - `get_available_models()`: 사용 가능한 모델 목록 제공
    - 현재 지원하는 모든 모델명 리스트 반환
  - `get_model_max_tokens(model_name)`: 모델별 최대 토큰 수 제공
    - 특정 모델의 최대 토큰 제한 조회 (기본값: 현재 설정된 모델)
    - 제한이 없는 모델의 경우 None 반환
- **의존 관계**: tiktoken 라이브러리에 의존
- **상세 설명**:
  - OpenAI 모델 계열(GPT 3.5, GPT 4 등)의 토큰화 규칙 준수
  - 모든 지원 모델에 대해 동일한 인터페이스 제공
  - 사용자 선택에 따른 모델 변경 가능성 고려
  - 토큰 계산 시 효율성 및 정확성 둘 다 고려한 설계
  - 파일 내용에 대한 토큰 계산 결과 캐싱 지원 (메인 윈도우 모듈과 연동)

#### filter.py
- **역할**: .gitignore 규칙 기반 파일/폴더 필터링
- **주요 기능**:
  - `GitignoreFilter` 클래스: .gitignore 파싱 및 필터링 관리
    - `__init__(root_dir)`: 지정된 루트 디렉토리에서 .gitignore 파일 찾아 로드
    - `_load_gitignore()`: .gitignore 파일 찾기 및 pathspec 객체로 변환
    - `should_ignore(path)`: 경로가 .gitignore 규칙에 의해 무시되어야 하는지 확인
    - `has_gitignore()`: 유효한 .gitignore 파일이 로드되었는지 확인
    - `get_gitignore_path()`: 로드된 .gitignore 파일 경로 반환
    - `filter_paths(paths)`: 경로 목록에서 .gitignore 규칙에 맞는 항목만 필터링
- **의존 관계**: pathspec 라이브러리에 의존
- **상세 설명**:
  - .gitignore 파일 자동 감지 및 로드:
    - 지정된 루트 디렉토리에서 .gitignore 파일 확인
    - 파일이 없거나 읽기 오류 시 적절한 처리
    - pathspec 라이브러리를 사용하여 효율적인 파일 경로 매칭 구현
  - 유연한 경로 처리:
    - 절대 경로와 상대 경로 모두 지원
    - 입력된 경로를 루트 디렉토리 기준 상대 경로로 자동 변환
    - OS 독립적 경로 구분자 처리 (Windows의 백슬래시, Unix의 슬래시)
  - 효율적인 패턴 매칭:
    - .gitignore 문법 완벽 지원 (*, **, ?, []등의 패턴)
    - 부정 패턴(!) 및 디렉토리 특정 패턴(/) 처리
    - 한 번 컴파일된 패턴을 재사용하여 성능 최적화
  - 강건한 오류 처리 및 로깅:
    - .gitignore 파일 로드 및 파싱 실패 시 정확한 오류 메시지 제공
    - 경로 매칭 중 발생한 예외 처리 및 로깅
    - 내부 상태 추적 및 디버깅 정보 제공
  - 다양한 활용 방식 지원:
    - 개별 경로에 대한 필터링 (`should_ignore()`)
    - 다수 경로 목록의 일괄 필터링 (`filter_paths()`)
    - 필터 상태 확인 (`has_gitignore()`)을 통한 조건부 적용 가능

#### output_formatter.py
- **역할**: 선택된 파일 및 폴더 정보를 정의된 출력 형식으로 변환
- **주요 기능**:
  - `generate_file_map(items, root_path, indent_size)`: 
    - 선택된 파일 및 폴더 구조를 `<file_map>` 형식으로 변환
    - 복잡한 파일 구조를 시각적으로 표현하기 위한 정렬 및 계층화 알고리즘 구현:
      1. 선택된 항목을 경로 기준으로 정렬
      2. 임시 디렉토리 트리 구조 구성 (중첩 딕셔너리 사용)
      3. 재귀적 순회를 통해 들여쓰기가 적용된 텍스트 형식으로 변환
    - 들여쓰기 크기를 인자로 받아 사용자 정의 가능 (기본값: 2 공백)
    - 빈 디렉토리도 적절하게 처리
    - 경로 구분자의 OS 독립적인 처리
  - `generate_file_contents(items, root_path)`: 
    - 선택된 파일들의 내용을 `<file_contents>` 형식으로 변환
    - 파일 확장자-언어 식별자 매핑 테이블 구현 (45개 이상의 확장자 지원)
    - 각 파일에 대해:
      1. 상대 경로 문자열로 변환 (OS 독립적 표현 위해 구분자 통일)
      2. 파일 확장자에 따른 적절한 언어 식별자 선택
      3. 파일 내용 읽기 시도 및 오류 처리
      4. 최종 포맷팅된 문자열 생성
    - 파일 읽기 오류 처리 개선:
      - 새로운 반환 타입(딕셔너리)에 대응하는 로직 추가
      - 오류 유형별 적절한 주석 메시지 생성
      - 로깅을 통한 오류 추적
  - `generate_full_output(items, root_path, indent_size)`: 
    - `<file_map>`과 `<file_contents>`를 모두 포함한 전체 출력 생성
    - 두 섹션 사이 빈 줄을 추가하여 가독성 향상
- **의존 관계**: core.file_scanner 모듈에 의존
- **상세 설명**:
  - 상대 경로 기반 파일 맵 생성
  - 언어별 코드 블록 형식 지원
  - 파일 확장자 기반 언어 자동 감지

### GUI 모듈 (src/gui/)

#### main_window.py
- **역할**: 애플리케이션 메인 윈도우 및 UI 구성
- **주요 기능**:
  - `MainWindow` 클래스: 메인 UI 윈도우
    - 트리 뷰, 버튼, 토큰 카운터 등 UI 구성
    - 폴더 선택 및 로드 기능
    - 체크박스 상태 관리 및 이벤트 처리
    - 백그라운드 토큰 계산 관리
  - `TokenizerThread` 클래스: 백그라운드 토큰 계산 스레드
    - `QThread`를 상속받은 전용 워커 스레드
    - 토큰 계산 및 진행 상황을 메인 UI에 보고하는 시그널 정의
    - 파일 내용 비동기 읽기 및 토큰화 처리
    - 스레드 안전한 중지 메커니즘 제공
- **의존 관계**: PySide6, core 모듈에 의존
- **상세 설명**:
  - QSplitter를 사용한 좌우 패널 분할
  - 좌측 패널에 파일/폴더 구조 트리 뷰
  - 우측 패널에 제어 버튼 및 정보 표시
  - 상태 표시줄 및 프로그레스 바 구현
  - 메뉴바 구현 (파일 메뉴, 도움말 메뉴)
  - 폴더 선택 다이얼로그 및 기본 폴더 로드 기능
  - 사용자 피드백을 위한 상태 메시지 표시
  - 파일 트리 뷰 기능:
    - 파일 유형별 다양한 아이콘 시스템:
      - 기본 아이콘 (폴더, 파일, 심볼릭 링크, 바이너리, 오류) 지원
      - 확장자별 특화 아이콘 (코드, 문서, 이미지 등) 제공
      - 파일 확장자-아이콘 매핑 테이블 (50+ 확장자 지원)
    - 파일 상태 시각적 표시:
      - 바이너리 파일: 회색 텍스트로 표시
      - 오류 파일: 빨간색 텍스트 및 경고 아이콘
      - 숨김 파일/폴더: 회색 텍스트로 표시
    - 상세 툴팁 정보 시스템:
      - 파일 경로, 크기, 언어 유형 표시
      - 심볼릭 링크 대상 경로 표시
      - 오류 파일의 상세 오류 정보 표시
      - 폴더의 항목 수 표시
    - 확장자-언어 매핑 시스템:
      - 70개 이상의 파일 확장자에 대한 프로그래밍 언어 매핑
      - C/C++, Python, JavaScript 등 주요 언어 파일 자동 인식
    - 트리 뷰 최적화 기능:
      - 균일 행 높이 설정으로 렌더링 성능 개선
      - 교차 행 색상 적용으로 가독성 향상
      - 열 너비 자동 조정
  - 체크박스 및 선택 관리:
    - 파일 및 디렉토리 아이템에 체크박스 구현
    - 체크박스 이벤트 처리 및 선택 항목 추적 (`_on_item_changed` 메서드)
    - 체크 상태 관리를 위한 구조화된 시스템:
      - `checked_items` 집합: 체크된 모든 항목(파일/디렉토리)의 경로 추적
      - `checked_files`: 체크된 파일 수 카운터 
      - `checked_dirs`: 체크된 디렉토리 수 카운터
    - 아이템 체크/해제 시 파일/디렉토리 구분하여 적절한 카운터 업데이트
    - 디렉토리 체크 상태 변경 감지 및 로깅 기능
    - 선택된 항목 경로 기반 토큰 계산 연동
    - 체크박스 상태 변경 시 실시간 UI 업데이트 (선택된 파일 수, 총 토큰 수)
    - 복사 버튼 활성화 상태 자동 관리 (체크된 파일이 있을 때만 활성화)
    - 부모/자식 체크박스 자동 선택/해제 기능:

      - `_update_parent_checked_state` 메서드: 자식 아이템 상태에 따른 부모 체크 상태 자동 결정

### 파일 시스템 스캔 및 처리 세부 구현

#### src/core/file_scanner.py
- **구현 세부사항**:
  - `scan_directory(root_path, follow_symlinks, include_hidden)`: 
    - 지정된 루트 경로부터 재귀적으로 모든 파일과 폴더를 스캔
    - 각 항목에 대해 경로, 상대 경로, 디렉토리 여부, 심볼릭 링크 여부, 숨김 여부 등의 정보를 수집
    - 권한 오류 등이 발생할 경우 오류 정보를 포함하여 접근 가능한 항목만 반환
    - `follow_symlinks` 옵션으로 심볼릭 링크를 따라갈지 결정
    - `include_hidden` 옵션으로 숨김 파일/폴더 포함 여부 결정
  - `is_hidden(path)`: 
    - OS에 독립적으로 파일/폴더의 숨김 여부 감지
    - Unix/Linux/Mac에서는 파일명이 '.'으로 시작하는지 확인
    - Windows에서는 시스템 API를 통해 파일 속성 확인
  - `is_binary_file(file_path)`: 
    - 파일의 앞부분을 읽어 널 바이트(0x00) 포함 여부로 텍스트/바이너리 판별
    - 파일이 비어있거나 존재하지 않는 경우 적절히 처리
    - 일반적인 바이너리 파일 확장자(.exe, .jpg, .pdf 등) 목록을 통한 빠른 식별
    - 확장자 기반 검사 후 바이트 검사를 통한 이중 확인 메커니즘 적용
  - `read_text_file(file_path)`: 
    - 텍스트 파일 내용을 문자열로 읽기
    - 다양한 인코딩을 체계적으로 시도 (UTF-8, UTF-8-sig, UTF-16, Latin-1, EUC-KR, CP949, Shift-JIS 등)
    - 파일 존재 여부, 바이너리 여부, 접근 권한, 인코딩 감지 실패 등 모든 오류 상황에 대한 상세한 오류 정보 제공
    - 오류 발생 시 단순 None 대신 구조화된 딕셔너리 반환 {'error': str, 'error_type': str, 'details': str}

#### src/gui/main_window.py 파일 시스템 관련 기능
- **구현 세부사항**:
  - `_open_folder_dialog()`: 
    - 사용자가 폴더를 선택할 수 있는 다이얼로그 표시
    - 선택된 폴더 경로가 있으면 `_load_folder()` 호출
  - `_load_folder(folder_path)`: 
    - 선택된 폴더 경로로 파일 시스템 스캔 수행
    - `scan_directory()` 함수 호출하여 파일/폴더 목록 가져옴
    - 스캔된 결과로 트리 뷰 모델 채우기 위해 `_populate_tree_view()` 호출
    - 오류 발생 시 사용자에게 메시지 박스로 알림
  - `_populate_tree_view(items)`: 
    - 스캔 결과를 트리 뷰 모델에 계층적으로 추가
    - 상대 경로 기반으로 부모-자식 관계 구성
    - 각 항목 유형(파일/폴더/심볼릭 링크/바이너리)에 맞는 아이콘 및 텍스트 설정
    - 추가 정보(접근 불가, 숨김 여부 등)를 툴팁으로 표시
  - `_toggle_hidden_files()`: 
    - 숨김 파일/폴더 표시 여부 토글 처리
    - 옵션 변경 시 현재 폴더를 다시 로드하여 UI 업데이트
  - `_init_icons()`: 
    - 파일, 폴더, 심볼릭 링크, 바이너리 파일, 오류 등을 나타내는 아이콘 초기화
    - 시스템 스타일의 표준 아이콘 활용

   - `is_binary_file` 함수가 각 파일의 바이너리 여부 판단
   - 첫 단계: 알려진 바이너리 확장자 목록과 비교 (빠른 판별)
   - 두 번째 단계: 확장자로 판별되지 않을 경우 파일 앞부분의 바이트 검사

3. **UI 표시**:
   - `_populate_tree_view` 메서드가 각 파일/폴더를 트리 뷰에 추가
   - 파일 유형에 따라 적절한 아이콘 할당 (폴더, 일반 파일, 바이너리 파일, 심볼릭 링크)
   - 바이너리 파일에 대해 회색 텍스트로 시각적 구분 추가
   - 각 항목 유형에 맞는 설명적 툴팁 추가
   - 바이너리 여부 플래그를 항목 데이터로 저장 (추후 토큰 계산에서 활용)

4. **사용자 상호작용**:
   - "숨김 파일/폴더 표시" 체크박스를 통해 숨김 항목 표시 여부 제어
   - 향후 체크박스 선택 UI에서 바이너리 파일 플래그를 참조하여 토큰 계산 제외

이 흐름을 통해 사용자는 파일 유형을 직관적으로 인식하고, 텍스트 파일만 효율적으로 선택하여 LLM 프롬프트에 포함시킬 수 있습니다. 바이너리 파일은 시각적으로 구분되어 토큰 계산이나 내용 추출에서 자동으로 제외됩니다.

## 파일 인코딩 처리 흐름

1. **파일 내용 읽기 요청**: 
   - `read_text_file` 함수가 파일 경로를 입력받아 내용을 읽기 시도
   - 먼저 파일 존재 여부와 바이너리 파일 여부를 확인하여 불필요한 읽기 시도 방지

2. **계층적 인코딩 처리**:
   - 첫 단계: 가장 일반적인 UTF-8 인코딩으로 읽기 시도
   - 실패 시: UTF-8-sig(BOM 포함), UTF-16 변형들, 다양한 지역별 인코딩(Latin-1, EUC-KR, CP949, Shift-JIS 등) 순차적 시도
   - 마지막 단계: ASCII 인코딩으로 최후 시도 (제한적이지만 기본 텍스트 확보)

3. **오류 처리 및 반환**:
   - 성공 시: 파일 내용 문자열 직접 반환
   - 실패 시: 구조화된 오류 정보를 담은 딕셔너리 반환
     - `file_not_found`: 파일이 존재하지 않는 경우
     - `binary_file`: 바이너리 파일로 감지된 경우
     - `access_denied`: 접근 권한이 없는 경우
     - `encoding_detection_failed`: 모든 인코딩 시도가 실패한 경우 (시도된 인코딩 목록 포함)
     - `read_error`: 기타 일반적인 파일 읽기 오류

4. **출력 포맷 통합**:
   - `generate_file_contents` 함수가 읽기 결과를 처리
   - 성공적인 읽기 결과는 그대로 파일 내용에 포함
   - 오류 정보는 해당 파일에 대해 설명적인 주석 형태로 변환 (예: "// 오류: PermissionError - 파일 읽기 권한이 없음")
   - 구조화된 오류 정보를 통해 사용자에게 명확한 문제 원인 제공

이 처리 흐름을 통해 다양한 인코딩의 텍스트 파일을 최대한 정확하게 읽어들이고, 읽기 불가능한 파일에 대해서는 명확한 오류 정보를 제공하여 사용자 경험을 개선합니다. 특히 다국어 환경의 소스 코드(한국어, 일본어, 중국어, 키릴 문자 등)에 대한 지원이 강화되었습니다.

## 주요 알고리즘 및 로직

### 파일 시스템 스캔 알고리즘
1. 루트 경로 유효성 검사
2. 루트 항목을 결과 목록에 추가
3. 재귀적으로 모든 하위 경로 탐색:
   - 각 항목마다 경로, 상대 경로, 디렉토리 여부, 심볼릭 링크 여부, 숨김 여부 정보 수집
   - 숨김 항목은 include_hidden 옵션에 따라 처리
   - 심볼릭 링크는 follow_symlinks 옵션에 따라 처리
   - 접근 불가 항목은 오류 정보와 함께 목록에 추가
4. 정렬된 항목 목록 반환

### 트리 모델 채우기 알고리즘
1. 트리 모델 초기화 및 토큰 캐시, 선택 목록 초기화
2. 경로별 아이템 사전 생성 (빠른 부모-자식 조회용)
3. 스캔 결과를 경로 기준으로 정렬
4. 루트 항목 생성 및 트리 모델에 추가
5. 각 항목마다 처리:
   - 부모 경로 확인하여 적절한 부모 아이템 찾기
   - 항목 유형에 따라 아이콘, 색상, 체크박스, 툴팁 설정
   - 오류 항목, 심볼릭 링크, 디렉토리, 일반 파일 각각 다르게 처리
   - 바이너리 여부 확인 및 표시
   - 파일 확장자 기반 언어 정보 및 적절한 아이콘 설정
6. 트리 전체 펼치기 및 열 너비 자동 조정

### 체크박스 상태 관리 알고리즘
1. 체크박스 상태 변경 감지 (itemChanged 시그널)
2. 변경된 아이템의 경로 조회
3. 체크 상태에 따라 선택된 항목 목록 업데이트
4. 선택된 항목의 총 토큰 수 계산:
   - 각 선택된 파일 내용 읽기 (캐시 확인 먼저)
   - 바이너리 파일 및 오류 상태 파일 제외
   - tokenizer를 통해 토큰 수 계산 및 캐싱
   - 총 토큰 수 합산
5. UI 정보 업데이트 (선택된 파일 수, 총 토큰 수, 복사 버튼 상태)

## 출력 포맷팅 모듈 세부 구현

### src/core/output_formatter.py
- **구현 세부사항**:
  - `generate_file_map(items, root_path, indent_size)`: 
    - 선택된 파일 및 폴더 구조를 `<file_map>` 형식으로 변환
    - 복잡한 파일 구조를 시각적으로 표현하기 위한 정렬 및 계층화 알고리즘 구현:
      1. 선택된 항목을 경로 기준으로 정렬
      2. 임시 디렉토리 트리 구조 구성 (중첩 딕셔너리 사용)
      3. 재귀적 순회를 통해 들여쓰기가 적용된 텍스트 형식으로 변환
    - 들여쓰기 크기를 인자로 받아 사용자 정의 가능 (기본값: 2 공백)
    - 빈 디렉토리도 적절하게 처리
    - 경로 구분자의 OS 독립적인 처리
  - `generate_file_contents(items, root_path)`: 
    - 선택된 파일들의 내용을 `<file_contents>` 형식으로 변환
    - 파일 확장자-언어 식별자 매핑 테이블 구현 (45개 이상의 확장자 지원)
    - 각 파일에 대해:
      1. 상대 경로 문자열로 변환 (OS 독립적 표현 위해 구분자 통일)
      2. 파일 확장자에 따른 적절한 언어 식별자 선택
      3. 파일 내용 읽기 시도 및 오류 처리
      4. 최종 포맷팅된 문자열 생성
    - 파일 읽기 오류 처리 개선:
      - 새로운 반환 타입(딕셔너리)에 대응하는 로직 추가
      - 오류 유형별 적절한 주석 메시지 생성
      - 로깅을 통한 오류 추적
  - `generate_full_output(items, root_path, indent_size)`: 
    - `<file_map>`과 `<file_contents>`를 모두 포함한 전체 출력 생성
    - 두 섹션 사이 빈 줄을 추가하여 가독성 향상
- **데이터 흐름**:
  1. 메인 윈도우에서 체크된 파일/폴더 항목 정보 수집
  2. 선택된 항목을 `output_formatter` 모듈에 전달
  3. 파일 맵과 파일 내용을 생성하여 하나의 문자열로 결합
  4. 결합된 문자열을 클립보드 유틸리티를 통해 시스템 클립보드에 복사

### src/gui/main_window.py의 복사 기능
- **구현 세부사항**:
  - `_copy_to_clipboard()` 메서드: 
    - 체크된 항목 정보 수집 및 정리
    - 복사 작업 수행:
      1. 체크된 항목 경로 목록(`self.checked_items`)에서 각 항목의 정보 추출
      2. 각 항목에 대해 절대 경로, 상대 경로, 디렉토리 여부 정보 구성
      3. 바이너리 파일 자동 필터링
      4. `generate_full_output()`을 호출하여 최종 출력 문자열 생성
      5. `copy_to_clipboard()`로 시스템 클립보드에 복사
    - 결과 피드백 제공:
      - 성공 시 "N개 파일 (M 토큰) 클립보드에 복사됨" 메시지 표시
      - 실패 시 오류 메시지 표시
    - 오류 처리 및 예외 상황 관리:
      - 예외 발생 시 로깅 및 사용자 알림
      - 메시지 박스를 통한 상세 오류 정보 제공
      - 항목 정보 수집 과정의 개별 오류 처리 (전체 작업 중단 방지)

### 연동 메커니즘
```
[사용자 동작] → 클립보드에 복사 버튼 클릭 → MainWindow._copy_to_clipboard()
    ↓
[체크된 항목 정보 수집] → output_formatter.generate_full_output() → 파일 구조 및 내용 포맷팅
    ↓
[결과 문자열] → clipboard_utils.copy_to_clipboard() → [시스템 클립보드]
    ↓
[상태 표시줄] → 복사 결과 피드백 → [사용자 확인]
```

이 구현을 통해 사용자는 파일 트리 뷰에서 원하는 파일과 폴더를 선택하고 "클립보드에 복사" 버튼을 클릭하면, 선택된 파일 구조와 내용이 LLM 프롬프트에 적합한 형식으로 클립보드에 복사됩니다. 이를 통해 코드베이스 맥락을 LLM에 쉽게 제공할 수 있으며, 적절한 언어 식별자 태그가 포함되어 코드 하이라이팅과 언어별 처리가 가능합니다.
